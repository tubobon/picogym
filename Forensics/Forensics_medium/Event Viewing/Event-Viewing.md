# Event-Viewing
## 解法
イベントビューアから3つのフラグの断片を見つける問題だった。\
問題文に

- 彼らはオンラインでダウンロードしたインストーラーを使ってソフトウェアをインストールしました
- インストールされたソフトウェアを実行しましたが、何も起きませんでした
- 今では、起動してログインするたびに、黒いコマンドプロンプト画面が素早く開閉し、パソコンは即座にシャットダウンします。

と書いてあるため、イベントIDをフィルタリングして見つける方法が便利

パソコンが即座にシャットダウンするということでシャットダウン要求であるイベントID1074をフィルタリングすると、コメントのところにbase64 encodeがされたflagの断片を発見（ここから残り2つもbase64だと推定した）\
続いて、インストーラにより製品がインストールされたことを伝えるイベントIDは1033なのだが、前回のflagの断片からbase64のflagであることがわかっているため、picoCTFのbase64が`cGljb0NURn`なのを利用し、この文字列を検索にかけたらflagの断片を発見\
最後にソフトウェアを実行したが何も起きなかったということで、初めはプロセスが作成された4688あたりを探していたが、それらしい痕跡を得られなかった。ということで、2つ目の断片のイベントに記載してあった`Totally_Legit_Software`というソフトウェア名を文字列検索にかけ、探索をしたところ、レジストリの変更イベント4657のところにflagの断片を発見

ソフトウェアを実行したが、何も起きなかったというのは、実は裏でレジストリの値を作成するようなプロセス（今回だとパソコンをシャットダウン）が動いていたため、そのイベント部分に断片があったわけだ

よって見つけた断片をつなげてflagを得た。`picoCTF{Ev3nt_vi3wv3r_1s_a_pr3tty_us3ful_t00l_81ba3fe9}`
